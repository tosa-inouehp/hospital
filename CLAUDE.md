## zyokusou.html（褥瘡管理支援ツール）

- **目的**：褥瘡の発生リスク評価・DESIGN-R® 2020評価・管理計画・経過記録のPDF出力
- **詳細仕様**：`hospital-apps/zyokusou/CLAUDE.md` を参照
- **現在のバージョン**：v1.4.0（2026-04-11）

### ステップ構成
1. STEP1：患者情報・報告者情報（必須バリデーションあり）
2. STEP2：Bradenスケール評価（6項目、最大23点）
3. STEP3：管理計画（寝具・体位変換・栄養管理・スキンケア・褥瘡の有無）
4. STEP4：褥瘡部位選択（カード式タップ選択）+ DESIGN-R® 2020評価
5. STEP5：患部写真（最大2枚・横向き推奨）・状態所見・治療方針・PDF出力

### PDF出力仕様
- **依存ライブラリ**：jsPDF 2.5.1・html2canvas 1.4.1（CDN経由）
- **1ページ目**：テキスト情報（DESIGN-R詳細表・Braden点数・管理計画）
- **2ページ目**：患部写真（縦2枚フル幅・高さ上限133.5mm/枚）
- **用紙**：A4縦（210×297mm）、マージン10mm

### DESIGN-R® 2020 スコア仕様（重要）
- **D（深さ）は合計スコアに加算しない**（重症度の参照指標のみ）
- 合計スコア = E + S + I + G + N + P（最大66点）

| 項目 | 選択肢・スコア |
|------|--------------|
| D（深さ） | d0=0 / d1=1 / d2=2 / D3=3 / D4=4 / D5=5 / DU=判定不能 |
| E（滲出液） | e0=0 / e1=1 / E3=3 / E6=6 |
| S（大きさ） | s0=0 / s3=3 / S6=6 / S8=8 / s9=9(36-64cm²) / S12=12(64-100cm²) / S15=15(≥100cm²) |
| I（炎症/感染） | i0=0 / i1=1 / i3C=3(臨界的定着) / I3=3 / I9=9 |
| G（肉芽組織） | g0=0 / g1=1(≥90%) / G3=3(50-90%) / G4=4(10-50%) / G5=5(<10%) / G6=6 |
| N（壊死組織） | n0=0 / n3=3 / N6=6 |
| P（ポケット） | p0=0 / p6=6 / P9=9 / P12=12 / P24=24 |

### localStorage キー
| キー | 内容 |
|------|------|
| `ulcer_draft_{timestamp}` | 下書きデータ（手動削除のみ・自動削除なし） |
| `ulcer_report_{id}_{date}` | 経時記録（PDF出力時に自動保存） |

## 開発ルール

- **単一HTMLファイル**で完結させること（HTML・CSS・JS はすべてインライン）
- **外部CDN依存は最小限**に（褥瘡ツールはjsPDF+html2canvasのみ許容）
- 患者氏名・IDなどの個人情報を**コードにハードコードしない**
- 作業は**一度に一つの変更**のみ行い、都度確認を取ること
